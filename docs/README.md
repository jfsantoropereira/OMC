# docs

## Purpose

This directory contains non-root documentation for OMC. At the current stage of the repository, that mainly means planning and design material that supports the eventual implementation.

## Structure

| Path | Purpose |
|---|---|
| [`planning/`](planning/) | Active planning documents for product shape, CLI design, UI changes, and implementation sequencing |

## Current State

These documents are **planning docs**, not implementation READMEs. Once OMC services, packages, or modules are actually built, durable technical knowledge should migrate into component `README.md` files and into [`SystemOutline.md`](../SystemOutline.md).
